Lithium Roadmap

Last Feature Update: 3/9/2022

These features are still being updated and improved, but carry the most significant functionality in the engine.

  • Filesystem Manager - Abstracts filesystem operations and handles loading some file formats.
  • Dynamic Shader Loader - Links related GLSL shaders together and binds them to C++ vertex and material shader classes and to input from material files.
  • Event-Driven GUI System - Interface set up in a way that mimics Java AWT.
  • C++ Scripting System - Developers may write C++ scripts to define the internal behavior of their application.
  • HDR and Post-Processing Support - Supports High Dynamic Range (HDR) rendering and post-processing effects such as Gaussian Blurs, Bloom and FXAA.
  • Material System - Handles loading material files which define a set of parameters to tell the engine how to draw an object. Supports albedo, specular, normal, ambient oucclusion and reflection mapping.
  • Shadow Mapping - Renders shadow maps which are used in shaders to create shadows in a scene.
  • Terrain Rendering - Rendered using a heightmap for vertex positions and splatmap for texture blending.
  • Particle Systems - Uses instanced rendering to be able to draw a large number of objects with little performance impact.
  • Authoring Tools - A set of tools for developers to create assets for their game outside of code.
  • Map System - Loads named binary map files, compiled from a text format, containing world geometry and entities defining behavior as well as automatically handling any loaded resources.

These features are currently being implemented and do not have full functionality.

  • Animation System - A system to automatically handle transformations of objects within a scene.
  • Cascaded Shadow Maps - Fragments closer to the camera use higher quality lightmaps compared to further ones to reduce perspective aliasing or shadow acne.
  • SSAO - Screen-Space Ambient Occlusion post-processing effect

These features I want to eventually add sometime in the future in no particular order.

  • Texture Array Support - To make more texture units taken by numerous material textures avaliable for other features.
  • Texture Animations - Textures change their look over time without C++ scripting.
  • PBR Rendering - Phyiscially-based rendering model.
  • Map Editor - A visual editor for features not related to C++ scripting or engine code.
  • Planar Reflections - Reflections on a plane using multiple cameras for reflection and refraction.
  • Post-Processing Effects - Adding more post-processing effects like screen-space reflections, depth-of-field, etc.
  • More Robust GUI System - The current GUI system can be confusing to use and should be able to support more GUI elements as well as be more trivial to use.
  • LScript Scripting - Support for users to use lscript files to control game behavior.
  • Support for Multiple Graphics APIs - DirectX 11, 12 and/or Vulkan.
  • Custom Libraries - Movement away from libraries like GLFW in favor of native window libraries to reduce overhead.
